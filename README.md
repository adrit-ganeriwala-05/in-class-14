## What This Project Does

This Flutter app demonstrates Firebase Cloud Messaging (FCM) integration across all three app lifecycle states. A cloud-delivered payload from Firebase Console updates the app UI in real time, proving that messages are received and handled correctly regardless of whether the app is open, backgrounded, or fully closed.

---

## FCM Handler Coverage

| State | Handler | What Happens |
|-------|---------|--------------|
| App open (foreground) | `onMessage` | UI updates immediately, status card changes |
| App backgrounded | `onMessageOpenedApp` | System notification shown, UI updates on tap |
| App terminated | `getInitialMessage()` | App cold-launches, payload applied on startup |

## Setup Instructions

### Prerequisites
- Flutter SDK installed
- Android emulator or physical device with Google Play Services
- Firebase project created at console.firebase.google.com

### 1. Clone the repo
```bash
git clone https://github.com/adrit-ganeriwala-05/in-class-14.git
cd in-class-14
```

### 2. Configure Firebase
Install the FlutterFire CLI and run configure:
```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=YOUR_FIREBASE_PROJECT_ID
```

This generates `lib/firebase_options.dart` and places `google-services.json` at `android/app/`.

### 3. Install dependencies
```bash
flutter pub get
```

### 4. Run the app
```bash
flutter run
```

### 5. Get your FCM token
The token appears in the debug console on startup:
```
🔑 FCM Token: eyJ...
```
Or tap **Copy Token for Firebase Console** in the app.

### 6. Send a test message
1. Go to Firebase Console → Engage → Messaging
2. Create a new campaign → Firebase Notification messages
3. Fill in title and body
4. Click **Send test message** → paste your token → Test

**Recommended test payload (add under Additional options → Custom data):**
```
asset  = promo
action = show_animation
screen = home
```

## Payload Keys

| Key | Values | Effect |
|-----|--------|--------|
| `asset` | `promo`, `alert`, `default` | Switches the displayed image/card color |
| `action` | `show_animation` | Triggers the pulse animation |
| `screen` | `home`, etc. | Logged as a deep-link hint |

## Notes

- `lib/firebase_options.dart` and `android/app/google-services.json` are excluded from the repo via `.gitignore` for security. Run `flutterfire configure` to regenerate them.
- `flutter_local_notifications` was intentionally excluded — it has a compilation bug with Android SDK 35 (`bigLargeIcon` ambiguous method). FCM message delivery works correctly without it.
- Tested on Android emulator (sdk gphone16k arm64, API 35).
